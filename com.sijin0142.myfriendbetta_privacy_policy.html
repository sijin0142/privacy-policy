5. Wi-Fi Device Transfer

The App provides a Wi-Fi Transfer feature that allows you to transfer all your betta data (database + photos) from one device to another on the same Wi-Fi network.

🔄 How It Works
  • The sending device acts as a local HTTP server (NanoHTTPD) and generates a QR code containing a temporary token and the local IP address.
  • The receiving device scans the QR code and downloads the data package directly over the local network.
  • The connection is secured by a one-time session token embedded in the QR code.
  • Data is compressed, checksummed (MD5), and verified before being applied on the receiving device.
🔒 Data transferred via Wi-Fi is sent directly between your two devices on your local network only. It is never routed through our servers or any external service. The temporary local server is closed immediately after the transfer completes or the screen is exited.
Warning: The Wi-Fi Transfer feature replaces ALL data on the receiving device. This action cannot be undone. Please ensure the data being received is correct before confirming.

6. Notifications

The App uses WorkManager to schedule local push notifications for water change and cleaning reminders. These notifications are generated entirely on-device and are not sent via any external push service (no FCM, no APNs).

  • Water Change Alert: Sent when the configured number of days has elapsed since the last water change (default: 7 days).
  • Cleaning Alert: Sent when the configured number of days has elapsed since the last cleaning.
  • Alert cycles can be configured in Settings → Alert Cycle.
  • Notifications can be disabled at any time via Android Settings → Apps → My Friend Betta❤ → Notifications, or by disabling the toggle in the app's Settings screen.
📱 All notifications are local and on-device. No external server is involved in sending notifications.

8. Data Security

  • All data is stored in the app's private internal storage (filesDir, Room DB), which is sandboxed and inaccessible to other apps.
  • Data is stored without additional application-level encryption, but is protected by Android's app sandbox. Device-level encryption applies based on your device settings.
  • Images are compressed (JPEG ~80% quality) before saving to reduce storage usage.
  • The temporary Wi-Fi transfer server uses a one-time session token and is immediately shut down after transfer or screen exit.
  • When the app is uninstalled, all app data in private storage is automatically deleted by the Android OS.

5. Wi-Fi 기기 간 데이터 전송

앱은 동일한 Wi-Fi 네트워크 내에서 모든 베타 데이터(DB + 사진)를 다른 기기로 전송할 수 있는 Wi-Fi 전송 기능을 제공합니다.

🔄 동작 방식
  • 보내는 기기가 로컬 HTTP 서버(NanoHTTPD)로 동작하며, 임시 토큰과 로컬 IP 주소가 포함된 QR 코드를 생성합니다.
  • 받는 기기는 QR 코드를 스캔하여 로컬 네트워크를 통해 데이터 패키지를 직접 다운로드합니다.
  • 연결은 QR 코드에 포함된 일회용 세션 토큰으로 보호됩니다.
  • 데이터는 압축 후 MD5 체크섬으로 검증하여 받는 기기에 적용됩니다.
🔒 Wi-Fi 전송으로 이동하는 데이터는 로컬 네트워크 안에서 두 기기 간에만 직접 전달됩니다. 외부 서버나 제3자 서비스를 경유하지 않습니다. 임시 로컬 서버는 전송 완료 또는 화면 이탈 즉시 종료됩니다.
주의: Wi-Fi 전송 기능은 받는 기기의 모든 데이터를 덮어씁니다. 이 작업은 취소할 수 없습니다. 최종 적용 전 수신 데이터를 반드시 확인하세요.

6. 알림 서비스

본 앱은 WorkManager를 이용하여 물갈이·청소 주기 알림을 로컬 푸시로 발송합니다. 이 알림은 기기 내부에서만 생성되며, FCM 등 외부 푸시 서비스를 사용하지 않습니다.

  • 물갈이 알림: 마지막 물갈이 기록 후 설정된 일수가 경과하면 발송됩니다 (기본값: 7일).
  • 청소 알림: 마지막 청소 기록 후 설정된 일수가 경과하면 발송됩니다.
  • 알림 주기는 설정 화면 → 알림 주기에서 변경할 수 있습니다.
  • Android 설정 → 앱 → 내 친구, 베타❤ → 알림에서 언제든 비활성화할 수 있습니다.
📱 모든 알림은 로컬에서 기기 자체적으로 발송됩니다. 외부 서버가 관여하지 않습니다.
